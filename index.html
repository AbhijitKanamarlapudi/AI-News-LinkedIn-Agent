Section I

Six Core
Capabilities

A complete end-to-end automation pipeline — from scraping the latest Anthropic research to publishing a structured, engagement-optimised LinkedIn post — with zero manual effort required after setup.

Fetch
01

Scrape & Rank

Scrapes Anthropic News and Research pages for the latest articles. Up to 10 links per source are fetched, parsed, and scored each run.

Score
02

Two-Tier Scoring

A relevance engine prioritises data-engineering content (Tier 1) over general AI updates (Tier 2), ensuring maximum audience impact every post.

Generate
03

AI-Crafted Posts

Generates structured posts with a keyword-matched hook, a "why it matters" section, a first-person takeaway, and up to 6 targeted hashtags.

Publish
04

Direct Publishing

Posts directly to your LinkedIn profile or company page via the official LinkedIn REST API using OAuth 2.0 — no browser automation needed.

Track
05

Deduplication

Every posted article URL is persisted to posted_articles.json. The same article is never published twice, automatically.

Extend
06

Fully Configurable

Schedule interval, scoring thresholds, news sources, hooks, takeaways, and hashtag patterns are all adjustable without rewriting core logic.

Posting

1 · Time Investment

With Agent

Generic take on the article, inconsistent format, hashtags chosen by gut feeling — impact varies wildly day to day.

Without Agent

With Agent

The built-in APScheduler runs on your configured cadence — whether you're available or not.

You only see articles that surface in your LinkedIn feed or what you happen to find when manually visiting the Anthropic website.
With Agent
Scrapes anthropic.com/news and anthropic.com/research simultaneously. Up to 10 articles per source scored and ranked per run — nothing is ever missed.

Section III

Setup Guide

Requirement 01

Python 3.9 or Later

The agent uses modern Python features unavailable in earlier versions. Check your version before starting — upgrade via python.org if needed.

python3 --version
Requirement 02

LinkedIn Developer App

Create an app at linkedin.com/developers. Request these OAuth 2.0 scopes: w_member_social, r_liteprofile, openid, profile. To post as a company page, also add w_organization_social.

Requirement 03

OAuth 2.0 Access Token

Use LinkedIn's OAuth 2.0 Token Generator in your app's Auth tab to generate a token with the scopes above. Tokens expire after 60 days — return to this step to refresh when needed.

Requirement 04

pip Package Manager

Ensure pip is available alongside your Python installation. All six runtime dependencies install in a single command with no compilation required.

pip --version
Step 01

Download the Project

Clone the project from GitHub and navigate into the project directory.

View on GitHub
git clone https://github.com/AbhijitKanamarlapudi/AI-News-LinkedIn-Agent cd AI-News-LinkedIn-Agent
Step 02

Create a Virtual Environment

Isolates the project's dependencies from your system Python. After activation you should see (.venv) at the start of your terminal prompt.

# Create python3 -m venv .venv # Activate (macOS / Linux) source .venv/bin/activate # Activate (Windows PowerShell) .venv\Scripts\Activate.ps1
Step 03

Install Dependencies

Installs all six runtime libraries: requests, beautifulsoup4, lxml, python-dotenv, APScheduler, and linkedin-api.

pip install -r requirements.txt
Step 04

Optional Dev Tools

Install pytest, black, and flake8 if you plan to run the test suite or contribute code changes.

pip install ".[dev]"
Config 01

Create Your .env File

Copy the provided example template and fill in your real values. The .env file is excluded from git via .gitignore — never commit it.

cp .env.example .env
Config 02

Minimal Working Configuration

At minimum you need your LinkedIn access token. Keep DRY_RUN=true while verifying your setup — nothing is published in this mode.

LINKEDIN_ACCESS_TOKEN=AQV...your_token LINKEDIN_DRY_RUN=true SCHEDULE_INTERVAL_HOURS=24
Config 03

Company Page (Optional)

To post as a LinkedIn Company Page instead of your personal profile, add the w_organization_social scope and provide your page ID.

LINKEDIN_PAGE_ID=123456789 # OR use the full URN directly LINKEDIN_PAGE_URN=urn:li:organization:123456789
Config 04

Author URN Resolution

The agent auto-resolves your identity via the /userinfo endpoint. If auto-resolution fails, set your URN manually. Find it by calling the userinfo API with your token.

curl -H "Authorization: Bearer YOUR_TOKEN" \ https://api.linkedin.com/v2/userinfo # Use the 'sub' field value: LINKEDIN_AUTHOR_URN=urn:li:person:<sub>
Step 01

Run a Dry-Run First

Verify your entire setup without publishing anything to LinkedIn. You should see article candidates, scores, and a full post content preview in the terminal output.

LINKEDIN_DRY_RUN=true python -m src.ai_agent
Step 02

Publish Your First Real Post

Set LINKEDIN_DRY_RUN=false in your .env, then run the agent. The article URL is saved to posted_articles.json after each successful post.

python -m src.ai_agent
Step 03

Run Once (No Internal Scheduler)

Ideal for use with macOS launchd, Linux cron, or GitHub Actions. Triggers a single post and exits immediately.

python -c "from src.ai_agent import \ AINewsLinkedInAgent; \ AINewsLinkedInAgent().run_once()"
Step 04

Schedule via macOS cron

Add to your crontab with crontab -e to post automatically at 9 AM every morning — no terminal session required.

0 9 * * * cd "/path/to/AI Agent LinkedIn" \ && .venv/bin/python -m src.ai_agent
Section IV

Troubleshooting
& Common Errors

403

Organization Permissions Error

You are posting as a company page but the w_organization_social scope is missing from your token, or LINKEDIN_PAGE_ID is not set in your .env. Ensure your LinkedIn account is an Admin of the company page.

→ Add w_organization_social scope · Regenerate token · Set LINKEDIN_PAGE_ID
401

Unauthorised — Token Expired

LinkedIn access tokens expire after 60 days. Return to the OAuth 2.0 Token Generator in your app's Auth tab, generate a new token with the same scopes, and update LINKEDIN_ACCESS_TOKEN in your .env.

→ Regenerate token every 60 days · Update LINKEDIN_ACCESS_TOKEN in .env
no
art.

No Articles Found

The Anthropic website may have changed its HTML structure, the request timed out, or all articles were filtered by the noise or scoring thresholds. Check your internet connection, then inspect NewsFetcher._extract_article_urls() for structural changes.

→ Check connectivity · Inspect _extract_article_urls() · Lower DATA_DEV_MIN_SCORE
dup.

Already Posted — 0 Posts Published

The top-ranked articles are already recorded in posted_articles.json. This is expected behaviour — the deduplication system is working correctly. Wait for Anthropic to publish new content, or delete the file for testing purposes only.

→ Wait for new articles · Testing only: rm posted_articles.json
dry

Posts Not Appearing on LinkedIn

The agent runs without errors but nothing is published to your profile. LINKEDIN_DRY_RUN is set to true (the default for safety). Set it to false in your .env to go live.

→ Set LINKEDIN_DRY_RUN=false in .env
mod.

ModuleNotFoundError

Dependencies are not installed, or Python is being invoked from outside the virtual environment. Activate the venv first, then re-run the install step before launching the agent.

→ source .venv/bin/activate · pip install -r requirements.txt
